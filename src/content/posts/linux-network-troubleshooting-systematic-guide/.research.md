# Research: Linux Network Troubleshooting – Systematic Diagnostic Workflow

## Topic Rationale
Network issues are among the most common production incidents. Existing articles cover OOM, CPU, disk, SSL, Nginx 502 — but no dedicated network troubleshooting guide exists. This fills a clear gap.

## Key Tools to Cover
1. **Layer-by-layer approach**: Physical → Link → Network → Transport → Application
2. **ping / traceroute / mtr**: Connectivity and path diagnostics
3. **ip / ss**: Interface state, routing table, socket inspection
4. **tcpdump / tshark**: Packet capture and analysis
5. **nslookup / dig**: DNS resolution issues
6. **curl / nc / telnet**: Application-layer connectivity testing
7. **ethtool**: Link-level diagnostics
8. **/proc/net/**: Kernel networking statistics
9. **ss -s / netstat -s**: Protocol statistics

## Common Scenarios
1. "Server is unreachable" — interface down, routing, firewall
2. "Connection refused" — port not listening, service down
3. "Connection timeout" — firewall drop, MTU, routing loop
4. "Slow network" — packet loss, DNS latency, TCP retransmissions
5. "DNS resolution failing" — /etc/resolv.conf, systemd-resolved
6. "Intermittent connectivity" — duplex mismatch, cable, ARP issues
