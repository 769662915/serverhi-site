## Selected Topic
Linux OOM Killer Troubleshooting — diagnosing and preventing Out-Of-Memory kills on production servers.

## Search Results Summary
1. eBPF can be used to monitor and troubleshoot container OOM kills (ebpfchirp.substack.com, dev.to)
2. OOM Killer is a Linux kernel mechanism that terminates processes when system runs critically low on memory
3. Common causes: memory leaks, misconfigured limits, lack of swap, runaway processes
4. Key diagnostic tools: journalctl -k, dmesg, /var/log/kern.log, free, top, htop
5. Prevention strategies: configure cgroup memory limits, set appropriate oom_score_adj, add swap, use monitoring

Practical guide covering:
