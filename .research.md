# Research Notes — Docker Log Rotation and Log Management

## Date: 2026-07-09
## Category: docker
## Working Title: "Docker Log Rotation Done Right: Stop Your Containers from Eating Your Disk"

## Key Points to Cover

### 1. The Problem
- Docker captures stdout/stderr from containers and stores logs via the logging driver
- Default json-file driver stores logs indefinitely with no rotation
- Production servers commonly hit disk-full issues from runaway container logs
- `docker logs` shows the symptom; `du -sh /var/lib/docker/containers/*/*.log` reveals the cause

### 2. Logging Drivers Overview
- json-file (default): writes JSON-formatted logs to host filesystem
- syslog: sends to syslog daemon
- journald: sends to systemd journal
- gelf: Graylog Extended Log Format
- fluentd: Fluentd collector
- awslogs/cloudwatch: AWS CloudWatch
- gcplogs: Google Cloud Logging
- splunk: Splunk
- local: minimal overhead, local file storage with rotation built-in

### 3. json-file Driver Rotation Config
- max-size: maximum log file size before rotation (e.g., "10m", "100m")
- max-file: maximum number of rotated log files to keep (e.g., "3")
- Can be set via daemon.json (global) or per-container (--log-opt)
- Compression is NOT built-in; need external logrotate or cron job

### 4. Daemon-Level Configuration (/etc/docker/daemon.json)
```json
{
  "log-driver": "json-file",
  "log-opts": {
    "max-size": "10m",
    "max-file": "3"
  }
}
```
- Requires Docker daemon restart: `systemctl restart docker`
- Existing containers NOT affected; must be recreated

### 5. Per-Container Configuration
```bash
docker run --log-opt max-size=10m --log-opt max-file=3 nginx
```
- Docker Compose:
```yaml
services:
  app:
    image: nginx
    logging:
      driver: json-file
      options:
        max-size: "10m"
        max-file: "3"
```

### 6. The local Driver Alternative
- Newer driver designed for minimal overhead
- Built-in rotation (max-size, max-file, compress)
- No external dependency
- Default: 20m max-size, 5 max-file
- Uses protobuf-based storage format

### 7. Aggregated Logging for Production
- Fluentd / Fluent Bit as sidecar
- Loki + Promtail (lightweight, Grafana-native)
- ELK stack (Elasticsearch + Logstash + Kibana)
- Quick setup with Docker Compose + Loki + Grafana

### 8. Practical Workflow / Commands
- Check current log sizes: `find /var/lib/docker/containers -name '*.log' -exec du -sh {} \; | sort -rh | head`
- Inspect container logging config: `docker inspect <container> | jq '.[0].HostConfig.LogConfig'`
- Truncate a runaway log: `truncate -s 0 /var/lib/docker/containers/<id>/<id>-json.log`
- Signal Docker to rotate: `docker run --log-opt max-size=1 ...` or send SIGHUP

### 9. Docker Compose Full Example
```yaml
version: "3.8"
services:
  web:
    image: nginx:alpine
    logging:
      driver: json-file
      options:
        max-size: "10m"
        max-file: "3"
        tag: "nginx-{{.Name}}"
  api:
    image: my-api:latest
    logging:
      driver: json-file
      options:
        max-size: "50m"
        max-file: "2"
```
